## 二、第2章 Java内存区域与内存溢出异常

### 运行时数据区域总览

- 程序计数器
- Java虚拟机栈
- 本地方法栈
- Java堆
- 方法区
- 运行时常量池
- 直接内存

### 程序计数器

- 线程私有
- 记录当前线程所执行字节码行号
- 线程切换后可恢复执行位置
- 唯一一个规范中没有规定任何OOM情况的区域

### Java虚拟机栈

- 线程私有
- 生命周期与线程相同
- 每个方法调用对应一个栈帧

### 栈帧结构

- 局部变量表

	- 基本数据类型
	- reference引用
	- returnAddress

- 操作数栈
- 动态连接
- 方法返回地址
- 附加信息

### 虚拟机栈异常

- StackOverflowError

	- 栈深度超出限制

- OutOfMemoryError

	- 无法申请足够栈空间
	- 无法创建新线程

### 本地方法栈

- 为Native方法服务
- HotSpot通常与虚拟机栈合并实现理解

### Java堆

- 线程共享
- 存放对象实例与数组
- GC管理的核心区域

### 堆的常见视角

- 分代视角

	- 新生代
	- 老年代

- Region视角

	- G1 / ZGC / Shenandoah更强调Region

### 对象创建过程

- 类加载检查
- 分配内存

	- 指针碰撞
	- 空闲列表

- 初始化零值
- 设置对象头
- 执行`<init>`

### 分配并发安全

- CAS + 失败重试
- TLAB线程本地分配缓冲

### 方法区

- 存放类型信息
- 运行时常量池
- 字段、方法、字节码描述
- 静态变量
- JIT编译后的代码缓存不属于方法区

### 永久代与元空间

- JDK 7：字符串常量池逐步移出永久代
- JDK 8：PermGen移除，改为Metaspace
- 元空间使用本地内存

### 运行时常量池

- 来源于Class文件常量池
- 内容：

	- 字面量
	- 符号引用
	- 运行期生成常量

- 典型案例：

	- `String.intern()`

### 直接内存

- 不属于JVM运行时数据区的一部分
- 常见于NIO
- 通过`DirectByteBuffer`间接使用
- 可能抛出OOM

### HotSpot对象探秘

- 对象头

	- Mark Word
	- 类型指针
	- 数组长度信息（数组对象）

- 实例数据
- 对齐填充

### 对象访问定位

- 句柄访问
- 直接指针访问
- HotSpot常用直接指针访问

### 常见内存溢出

- Java堆溢出
- 虚拟机栈与本地方法栈溢出
- 方法区 / 元空间溢出
- 运行时常量池溢出
- 直接内存溢出

## 三、第3章 垃圾收集器与内存分配策略

### 对象是否已死

- 引用计数法

	- 实现简单
	- 难以解决循环引用

- 可达性分析

	- 主流JVM采用

### GC Roots

- 虚拟机栈中的引用
- 方法区中类静态属性引用
- 方法区中常量引用
- 本地方法栈中JNI引用
- 被同步锁持有的对象
- JVM内部引用

### 引用类型

- 强引用
- 软引用
- 弱引用
- 虚引用

### 对象死亡判定补充

- 第一次标记：不可达
- 筛选是否需要执行`finalize()`
- 第二次标记：仍不可达则回收
- `finalize()`只会被系统自动调用一次

### 方法区回收

- 废弃常量回收
- 类型卸载

	- 该类所有实例已被回收
	- 加载该类的类加载器可被回收
	- 对应`java.lang.Class`对象无处引用

### 垃圾收集算法

- 标记-清除

	- 简单
	- 碎片化严重

- 标记-复制

	- 适合新生代
	- 空间换时间

- 标记-整理

	- 适合老年代

- 分代收集理论

	- 弱分代假说
	- 强分代假说
	- 跨代引用假说

### HotSpot算法细节实现

- 枚举根节点
- 安全点

	- 抢先式中断
	- 主动式中断

- 安全区域
- 记忆集与卡表
- 写屏障
- 并发可达性分析

### 经典垃圾收集器

- Serial
- Serial Old
- ParNew
- Parallel Scavenge
- Parallel Old
- CMS

	- 优点：低停顿
	- 缺点：

		- 并发阶段对CPU敏感
		- 浮动垃圾
		- 内存碎片

- G1

	- Region化
	- 追求可预测停顿
	- Mixed GC

### 低延迟垃圾收集器

- Shenandoah

	- 并发压缩
	- 读屏障

- ZGC

	- 染色指针
	- 读屏障
	- 极低停顿

### 如何选择收集器

- 吞吐量优先

	- Parallel Scavenge / Parallel Old

- 响应时间优先

	- G1
	- ZGC
	- Shenandoah

- 单核/小堆/Client场景

	- Serial

### 内存分配与回收策略

- 对象优先在Eden分配
- 大对象直接进入老年代
- 长期存活对象进入老年代
- 动态对象年龄判定
- 空间分配担保
