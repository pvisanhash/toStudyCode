

简单说：**L4 和 L7 指的是负载均衡工作在网络模型的不同层级（OSI模型）**，决定了它“看得懂多少数据、能做多复杂的决策”。

---

## 一、四层负载均衡（L4）

👉 工作在 **传输层（TCP/UDP）**

### 核心特点

- 只看“网络层信息”
    
    - 源 IP、目标 IP
        
    - 源端口、目标端口
        
- **不关心具体业务内容（HTTP、SQL等）**
    
- 转发速度快、性能高
    

### 举个直观例子

就像快递分拣：

> 只看地址（IP+端口），不拆包裹

### 工作方式

- 客户端 → 负载均衡器
    
- 根据 IP/端口 → 转发到后端服务器
    
- 不解析请求内容
    

### 常见技术

- **LVS（你当前看的这个）**
    
- 四层模式的 HAProxy
    

### 优点

- 性能极高（内核级、转发快）
    
- 延迟低
    
- 适合高并发（百万级QPS）
    

### 缺点

- 不智能（看不懂业务）
    
- 不能根据 URL、Header 做路由
    

---

## 二、七层负载均衡（L7）

👉 工作在 **应用层（HTTP/HTTPS等）**

### 核心特点

- 能解析完整请求内容
    
    - URL
        
    - Header
        
    - Cookie
        
    - 请求体
        
- 可以做“业务级路由”
    

### 举个直观例子

还是快递：

> 不仅看地址，还拆开看内容，再决定送去哪

### 工作方式

- 客户端 → 负载均衡器
    
- 解析 HTTP 请求
    
- 根据规则转发
    

例如：

```
/api → 服务A
/static → 服务B
```

### 常见技术

- Nginx
    
- HAProxy（7层模式）
    
- Spring Cloud Gateway
    

### 优点

- 非常灵活（智能路由）
    
- 支持：
    
    - URL 路由
        
    - 灰度发布
        
    - A/B 测试
        
    - 鉴权/限流
        

### 缺点

- 性能比 L4 低（需要解析协议）
    
- 延迟略高
    

---

## 三、L4 vs L7 对比（重点）

|对比项|L4（四层）|L7（七层）|
|---|---|---|
|工作层|传输层|应用层|
|处理内容|IP + 端口|HTTP内容|
|是否解析业务|❌|✅|
|性能|⭐⭐⭐⭐⭐|⭐⭐⭐|
|灵活性|⭐⭐|⭐⭐⭐⭐⭐|
|典型组件|LVS|Nginx|

---

## 四、你可以这样理解（面试级总结）

👉 一句话：

> **L4 是“按地址转发”，L7 是“按内容转发”**

---

## 五、实际架构中怎么用？

真实生产中一般是：

```
用户
  ↓
L4（LVS，抗流量）
  ↓
L7（Nginx，做业务路由）
  ↓
应用服务
```

👉 原因：

- L4 抗压（扛流量）
    
- L7 做智能分发
