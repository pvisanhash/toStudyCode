# 一、先搞清 JVM 到底在解决什么（半天内）

别一上来就学细节，先建立“地图”，否则后面全是碎片。

你需要理解 JVM 的 3 件事：

### 1. JVM 做了什么

- Java代码 → 字节码 → JVM执行
    
- 跨平台的核心：**字节码 + JVM实现**
    

### 2. JVM 三大核心模块

- 类加载（ClassLoader）
    
- 内存模型（Runtime Data Area）
    
- 执行引擎（解释器 + JIT）
    

👉 这一阶段建议快速过一遍：

- 深入理解Java虚拟机（不用细读，先扫目录+核心章节）
    

---

# 二、优先打穿“内存模型 + GC”（最关键，1~2天）

这是 JVM 的核心，占面试 + 实战的 70%

## 1. JVM 内存结构（必须掌握）

重点理解：

- 堆（Heap）
    
- 栈（Stack）
    
- 方法区（Metaspace）
    
- 程序计数器
    

搞清几个问题：

- 对象存在哪？
    
- 方法调用怎么压栈？
    
- 为什么会 OOM？
    

---

## 2. 垃圾回收（GC）——重点中的重点

你要掌握的不是“所有算法”，而是：

### 核心算法（理解原理）

- 标记清除
    
- 复制
    
- 标记整理
    

### 常见收集器（必须会对比）

- Serial
    
- Parallel
    
- CMS（已淘汰但面试常问）
    
- G1（重点）
    
- ZGC（了解）
    

👉 重点理解：

- Minor GC / Full GC 区别
    
- Stop The World 是什么
    
- G1 为什么适合大内存

# 七、最关键：别只看，一定要“动手”

给你一个最有效的方法：

## 1. 自己制造问题

比如：

```java
List<byte[]> list = new ArrayList<>();
while (true) {
    list.add(new byte[1 * 1024 * 1024]);
}
```

👉 你去观察：

- GC日志
    
- 堆变化
    
- OOM发生过程
    

---

## 2. 配合参数玩

```bash

-Xmx100m
-XX:+PrintGCDetails
```
